SHOP NOW Button: A/B Test,
or Something More?

Python Pandas SciPy Chi-Square Test Regex

Eniac wanted to know whether tweaking their "SHOP NOW" button — color, copy, or both — would improve click-through rate. The obvious framing is "A/B test it." But with two variables and four combinations, this is technically a 2×2 multivariate test, not a simple A/B test — and that distinction turned out to matter, because the two changes didn't just add up, they fought each other. We ran this under our tutor's supervision, and the numbers below are the ones we stand behind.

The Brief

Four button versions, one question

Eniac's "SHOP NOW" button was tested in four variants across roughly 100,000 visits over two weeks, tracking click-through rate (CTR) as the primary metric.

Not quite A/B

Two independent variables — button color (original vs. red) and button copy ("SHOP NOW" vs. "SEE DEALS") — were combined into four versions. That's a 2×2 factorial design: a multivariate test (MVT) wearing an A/B test's clothes. The difference matters because an MVT can reveal interaction effects — cases where two changes don't just add together, but actively work against each other. A simple A/B test would never catch that.

Getting Started

Key design questions before launching

Before touching the data, we had to answer a handful of design questions — the kind that determine whether the results mean anything at all.

📊 A/B or Multivariate?
2 variables × 4 combinations = a 2×2 factorial design, not a simple A/B test.
🎯 What metric?
Primary KPI: click-through rate — a discrete, binomial metric. That points straight to a chi-square test.
⚖️ How to split traffic?
Random assignment per visitor, so exposure is equal and unbiased across all four versions.
🔬 Guarding against chance?
Significance threshold set at α = 0.05 before launch — not after peeking at results.
⏱️ How long to run?
Ran for the full ~2 weeks needed to hit sample size — no early stopping.
🛒 What else could we test?
Hero section, search, and pricing UX all get more traffic — but one variable at a time keeps results interpretable.

VerdictA p-value this small leaves no doubt — the four versions do not perform equally. Something about color or copy is genuinely driving the difference.
02

Copy wins, color backfires

Changing only the text to "SEE DEALS" (Version C) nudged CTR slightly above control. Changing only the color to red (Version B) — the "obvious" high-contrast move — actually made things worse.

VerdictThe common assumption that red buttons convert better didn't hold up. Copy mattered; color, on its own, hurt.
03

Combining both changes made things worse, not better

Version D — red color and new copy — didn't land between B and C. It scored lowest of all four versions, well below even Version B alone.

VerdictThis is the interaction effect an A/B test would have missed entirely: the two changes don't add, they actively undercut each other.
04

Pairwise comparisons, Bonferroni-corrected

To see exactly which versions differ from which, we ran pairwise chi-square tests with a Bonferroni-corrected threshold (α = 0.05 / 6 = 0.00833) to avoid false positives from multiple comparisons.

So What

What we recommended

Roll out Version C. "SEE DEALS" is a simple copy change with the best measured CTR — low effort, real (if modest) upside.
Don't ship Version D. The combination underperforms every other version, including the ones it was built from.
Investigate why red hurt. It contradicts the usual "high-contrast CTA" assumption — worth testing other colors (green, orange, blue) rather than writing off color changes entirely.
Keep testing combinations, not just single changes. The interaction effect here only showed up because we tested a full 2×2 grid instead of one variable at a time.
Conclusion

My take

"The statistics weren't the hard part — the hard part was noticing that this wasn't a simple A/B test in the first place. Once we named it correctly as a 2×2 design, the interaction effect between color and copy was the most interesting finding in the whole project, and it's exactly the kind of thing a naive A/B mindset would have missed." — Iliya, on finishing this project
